1. Problem Statement

The "Vibe Coding" Context Drift:

As solo developers and teams delegate massive execution blocks to AI coding assistants (such as Cursor, Claude Code, or Windsurf), a critical gap emerges: requirement drift.

  • Scattered Knowledge: Crucial product goals, edge cases, architectural decisions, and API agreements get buried in deep AI chat histories or temporary IDE context.
  • Limited Memory: Standard AI agents operate within limited context windows and cannot preserve long-term product alignment as codebases scale.
  • Loss of Engineering Control: Developers gradually lose track of how AI-generated modules interact with the original product roadmap, resulting in bloated scopes, broken critical paths, and wasted development cycles.

2. Solution

AI Product Manager is a conversational 6-expert agent system that translates fuzzy product ideas into a structured, machine-readable Product Requirements Document (PRD), which is then compiled into a live Scrum board with capability-calibrated deadlines, dynamic progress telemetry, and auto-generated developer prompts.

By grounding all agent reasoning in a central database schema rather than chat context alone, it guarantees long-term project alignment for solo builders and teams.

3. Agent Architecture (6-Stage Pipeline)

We built 6 specialized agents that orchestrate a sequential chain-of-thought pipeline. Each expert operates with a custom system prompt, discrete goals, and typed JSON handoffs.

Step Agent / Expert Role Primary Focus Methodology & Output Format
1 Senior Business Requirements Analyst Intent Capture Extracts the core Intent Pair (Action + Purpose). Uses funnel-type questioning to refine vague briefs; rejects assumptions and calibrates via analogy scenarios.
2 Strategic Intelligence Analyst Task Profile Builds a 4D Task Profile (Background, Constraints, Expected Outcome, Capability). Conducts BEI (Behavioral Event Interview) and Inference Chain stakeholder analysis.
3 Chief Product Architect User Journey & Backlog Maps a linear User Journey and decomposes it into functional nodes. Categorizes the product backlog using strict MoSCoW prioritization (P0 to P3).
4 Technical Solutions Architect Specs & Technical Docs Executes 3-round progressive elaboration (Requirements to Feasibility to Goal Alignment). Generates technical specs and ready-to-use Cursor/Claude Code system prompts.
5 Senior Delivery Manager Scheduling & Effort Applies capability-calibrated estimation multipliers (Proficient: 0.8x, Basic: 1.2x, New: 1.5x) plus a mandatory 20% buffer. Sequences milestones.
6 Agile Delivery Coach Deviation-Driven Control Continuous evaluation loop (triggers every 5% of sprint progress). Computes the Efficiency Index (EI) and applies dynamic adjustment paths to prevent delay.

4. Technical Specifications & Tool Use

4.1 AI & Agent Orchestration

  • Primary LLM: DeepSeek V4 Series (deepseek-v4-flash for fast iterations, deepseek-v4-pro for deep reasoning/thinking paths).
  • Fallback LLM: OpenAI models (seamless automatic failover routing).
  • Validation: All LLM outputs are structured using JSON mode and strictly validated server-side using Zod schemas before database entry.
  • Integration: Fully supports the Model Context Protocol (MCP) via a custom JSON-RPC /api/mcp endpoint to export these expert capabilities to external desktop tools.

4.2 Data & Persistence

  • Database Engine: PostgreSQL hosted on Serverless Neon DB.
  • ORM: Prisma 7 with Native PG Driver Adapter (@prisma/adapter-pg + pg client pool).
  • State Management: All project backlogs, sprint tasks, and agent session checkpoints reside entirely in the cloud database. The filesystem is kept completely stateless, allowing rapid, secure deployments.

4.3 Developer Handoff & UI/UX

  • One-Click Markdown Export: Generates clean project briefs, functional specifications, and developer system prompts pre-formatted for Cursor, Windsurf, or Claude Code.
  • UI Themes: Features a sleek dark Liquid Glass layout (custom CSS glassmorphism, responsive grids, and micro-interactions) with a switchable retro Sketch theme for paper-analog lovers.
  • Scrum Workspace: Embeds a live Kanban board, sprint logs, and a dynamic mathematical Burndown Chart (utilizing projected remaining effort to estimate true sprint completion trends).
